COMIT - Competitive IT Events

Abstract

COMIT stands for "Competitive IT Events" a Web service-oriented application which provides means to explore, filter, visualize, and update information regarding the participants to various IT competitive events (e.g., programming or algorithm-related competitions, Web challenges, hackathons, etc.) and their submitted solutions.

Furthermore, the system provides useful recommendations about alternative solutions, additional tips & tricks, similar problems and events, tutorials, on-line courses, public code repositories, interesting persons (mentors, winners, organizing entities, sponsors, and others).

The web application is built with Java on the server-side, using Spring Boot as a framework and PostgreSql with pgAdmin4; as for the client-side, the application is build with Angular 8. Moreover, the application is built upon MVC architecture.

Development methodology

Waterfall methodology is a linear project management approach, where stakeholder and customer requirements are gathered at the beginning of the project, and then a sequential project plan is created to accommodate those requirements.

The 5 phases used in the develompent process of this project are:

  1. Requirements: The key aspect of waterfall is that all customer requirements are gathered at the beginning of the project, allowing every other phase to be planned without further customer involvement until the product is complete. It is assumed that all requirements can be gathered at this phase.
  2. Design: The design phase is best broken up into logical design and physical design subphases. The logical design subphase is when possible solutions are brainstormed and theorized. The physical design subphase is when those theoretical ideas and schemas are made into concrete specifications.
  3. Implementation: The implementation phase is when programmers assimilate the requirements and specifications from the previous phases and produce actual code.
  4. Verification: This phase is when the customer reviews the product to make sure that it meets the requirements laid out at the beginning of the project. This is done by releasing a completed product to the customer.
  5. Maintenance: The customer is regularly using the product during the maintenance phase, discovering bugs, inadequate features and other errors that occurred during production. The production team applies these fixes as necessary until the customer is satisfied.

Main functionalities

The main functionalities and feautures of the COMIT Web Application are split on:

  • Guest functionalities - mainly login and register;
  • User functionalities - how the standard will interact with the application and what functionalities are available to such users;
  • Admin functionalities - mainly the functionalities which regard the administration and maintance of the application.

Guest functionalities

The main functionalities available to guest users are:

  • A guest would be able to create a new account through the "Sign Up" page;
  • A guest would be able to login through the "Login" page;
  • A guest can visualize all the available events and challenges, but can't provide solutions to challenges.

The functionalities provided to the guest user are quite limited, as other interactions require informations about the user.

User functionalities

The main functionalities available to standard users are:

  • A standard user can visualize all the available events;
  • A standard user can visualize all the events available challenges;
  • A standard user can filter events by:
    • Event type;
    • Technology;
    • Knowledge level;
    • Status.
  • A standard user can provide solutions to active challenges;
  • A standard user can visualize the alternative provided solutions.

Standard user have a wider range of functionalities compared to guest users.

Admin functionalities

The admin has mainly the administrative functionalities required to mantain the application and ensure its normal flow as well as provide new content. The available functionalities are:

  • Add new events;
  • Add challenges to events;
  • Provide alternative solutions to challenges.

Technologies

The main technologies used for the COMIT Web application are:

  • Backend: Java + Maven + Spring Boot
  • Frontend: Angular 2+ with Bootstrap (CSS framework)
  • DB: PostgreSQL

Install

In order to run this application on your local machine, you must download a copy of this project and import it in your IDE (either Eclipse/ VS Code / IntelliJ). Using the source code provided in this repository, you can import the project by cloning it with the project's URL, or you can download the ZIP folder and import the unzipped project.

Once you have imported the project, before running it, you must first create an empty database in pgAdmin. After creating the database, you must got to ApplicationProperties.java file and change the properties to match your local database, including database name, database URL and password.

Conclusions

In conclusion, COMIT is a web application, designed and developed for programmers of all levels, where they can practice their coding skills faster and more efficient. The platform offers a considerable number of IT competitions, each having multiple challenges. Users can participate in these challenges and submit their solutions, using whatever programming language and difficulty level that best suits them.

The design of the application was made using Balsamiq Wireframes Cloud tool; the back-end was built on Java, using Spring Tool Suite and the front-end was built on JavaScript, CSS and HTML, using Angular CLI Framework.
